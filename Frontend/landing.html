Cómo funciona

De código a insight en segundos

El pipeline de análisis completo, explicado.

01
Cargá tu repo
Subí archivos .py, un .zip, o pegá un dump. El launcher acepta directorios directamente desde la terminal.
02
Parseo AST
Cada archivo se parsea con el módulo `ast` de Python. Errores de sintaxis se reportan sin detener el análisis.
03
Análisis multi-capa
Dependencias, estructura (clases/funciones/decoradores), call graph con líneas, y grafo de nodos/aristas.
04
Dead code scoring
Cada función y método recibe un score basado en callers, tipo, decoradores y convenciones de nombre.
05
Visualización
El grafo se renderiza en canvas 2D con force layout. Seleccioná nodos para ver detalle completo.
API

REST API documentada

Integrá el análisis en tus propias herramientas. Documentación automática en /docs.

MétodoRutaDescripción
GET/healthHealth check del servidor
POST/analyze/dumpAnaliza un dump de texto (JSON body)
POST/analyze/uploadSube archivos .py (multipart)
POST/analyze/zipSube un .zip con archivos .py
POST/traceabilityTrazabilidad de un archivo específico
# Subir archivos .py
curl -X POST http://127.0.0.1:8000/analyze/upload \
  -F "files=@mi_modulo.py" \
  -F "files=@otro.py"

# Subir ZIP
curl -X POST http://127.0.0.1:8000/analyze/zip \
  -F "file=@mi_proyecto.zip"

# Pegar dump
curl -X POST http://127.0.0.1:8000/analyze/dump \
  -H "Content-Type: application/json" \
  -d '{"content": "..."}'
import requests

# Subir archivos .py
files = [
    ("files", open("mi_modulo.py", "rb")),
    ("files", open("otro.py", "rb")),
]
r = requests.post("http://127.0.0.1:8000/analyze/upload", files=files)
analysis = r.json()

# Acceder al resumen
print(analysis["summary"])

# Dead code de alto riesgo
for fp, items in analysis["dead_code"]["items"].items():
    high = [i for i in items if i["score"] >= 0.7]
    if high:
        print(f"{fp}: {len(high)} elementos dead code")
